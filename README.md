# Nexus - платформа обмена навыками

## О проекте
Веб-платформа для горизонтального обмена знаниями и навыками между пользователями университетов и частными лицами в формате «равный равному» через синхронные видео-комнаты с системой взаимной оценки.

## Стек технологий
* **UI/UX Design & Prototyping: Figma** разработка интерактивных высокодетализированных макетов, создание единой дизайн-системы, UI-Kit и компонентной сетки интерфейса платформы. 
* **Miro:** проектирование пользовательских путей.

* **Frontend:** HTML5, CSS3, JavaScript (сгенерирован и оптимизирован с помощью ИИ Qwen).
* **Backend & Database:** Supabase (PostgreSQL, Supabase Auth для авторизации). 
* **Безопасность: Row Level Security (RLS)** для изоляции данных пользователей.
* **Таск-трекер и управление** WEEEK (Scrum/Kanban).
* **GitHub:** правление версиями, хостинг исходного кода проекта, проведение код-ревью и контроль стабильности релизных веток приложения.

## Ключевые фичи
* **Авторизация:** Безопасный вход в систему через почту.
* **Профили пользователей:** Анкета студента с указанием того, чему он может научить и чему хочет научиться.
* **Экономика Skill Coins:** Логика транзакций (списание 5 коинов у ученика, начисление 3 коинов ментору с учётом налога платформы).
* **Система поиска:** Нахождение менторов по навыкам.

## Команда проекта
### 1. Алиса
* **Роли:** Информационный архитектор, UX-исследователь.
* **Зона ответственности:**
    * **Проектирование структуры:** Разработка информационной архитектуры платформы (карта сайта, логика переходов между экранами).
    * **Анализ сценариев:** Проектирование логики пользовательских путей (*User Flow*) - как студент находит ментора и совершает сделку.
    * **Подготовка компонентов:** Техническая верстка и «полировка» макетов в Figma, выравнивание элементов по сетке (Pixel-Perfect), дополнение контентом.

---

### 2. Милена
* **Роли:** Ведущий дизайнер интерфейсов, Концептуальный UI-художник, Frontend-разработчик.
* **Зона ответственности:**
    * **Визуальный стиль:** Создание единой дизайн-системы и *UI-kit* проекта (фирменные цвета, шрифты, стили кнопок, иконки).
    * **Чистовой дизайн:** Отрисовка высокодетализированных финальных макетов всех экранов платформы в *Figma*.
    * **Генерация базовой структуры фронтенда через ИИ:** С использованием языковой модели Qwen преобразовала готовые Hi-Fi прототипы из Figma в рабочий HTML/CSS/JS код. В модель прописывались детальные цепочки сценариев использования, на основе которых Qwen генерировал начальную структуру сайта: визуальное оформление, цветовую схему, шрифты и базовую вёрстку страниц. На этом этапе была создана основа интерфейса без глубокой проработки клиентской логики.

---

### 3. Олеся
* **Роли:** *Project Manager/Team Lead*, Архитектор баз данных, Фронтенд-разработчик
* **Зона ответственности:**
    * **Управление проектом:** Координация работы команды, ведение таск-трекера, контроль дедлайнов и организация *Scrum*-спринтов.
    * **Проектирование и администрирование БД:** Разработка реляционную структуру базы данных в Supabase (таблицы profiles, universities, institutes, group, lessons, skills_list). Настройка связи (Foreign Keys), триггеры для автоматического создания профилей и Row Level Security (RLS) политики для разграничения прав доступа.
    * **Доработка и интеграция фронтенда:** После генерации базовой структуры сайта через Qwen выполнила полную доработку фронтенда: довела вёрстку до финального состояния, реализовала недостающую клиентскую логику (работа с формами, динамическое отображение данных, обработка пользовательских действий), интегрировала фронтенд с базой данных Supabase через JavaScript SDK, обеспечив полноценную работу всех сценариев: регистрации, поиска, записи на уроки, системы коинов и чатов.
    * **Документация:** Участие в подготовке технической документации проекта (описание продуктовых гипотез, метрик успеха, MoSCoW и дорожной карты).

---

### 4. Настя
* **Роли:** Системный аналитик, Технический писатель, Продуктовый маркетолог (*Product Marketer*).
* **Зона ответственности:**
    * **Анализ требований:** Формализация идей команды, сбор функциональных требований и составление Технического задания (ТЗ).
    * **Создание документации:** Полное написание, структурирование и оформление отчетной текстовой документации проекта.
    * **Презентация и упаковка:** Разработка визуальной концепции и структуры финальной презентации, подготовка проекта к защите.
