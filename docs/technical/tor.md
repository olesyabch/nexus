# ТЕХНИЧЕСКОЕ ЗАДАНИЕ
## Платформа обмена навыками «НЕКСУС»
**Версия:** 1.0 | **Дата составления:** март 2026 г.

---

## 1. Краткое описание проекта

«Нексус» — веб-платформа горизонтального обмена знаниями и навыками в формате «равный равному» (P2P). Платформа адресует три ключевых барьера современного EdTech-рынка:

* **Проблема 1. Высокий финансовый барьер** До 40% студентов не могут позволить себе регулярное платное наставничество, а начинающие специалисты не могут монетизировать навыки из-за высоких комиссий платформ.
* **Проблема 2. Кризис вовлеченности** Средний показатель завершаемости курсов на Coursera, Stepik и аналогах — ниже 10% из-за отсутствия живого контакта и обратной связи.
* **Проблема 3. Асимметрия навыков** Внутри университетов существует явный дисбаланс компетенций, но прямой (двусторонний) бартер навыков не работает — найти человека с «зеркальными» потребностями вручную практически невозможно.

### Решение
«Нексус» решает эти проблемы за счет:
* Внутренней геймифицированной экономики **Skill Coins** (без реальных денег), устраняющей финансовый барьер;
* Синхронных **P2P-сессий (live)**, заменяющих пассивный контент живым взаимодействием;
* **Многостороннего мэтчинга** по навыкам, ликвидирующего проблему «холодного старта» при поиске партнера.

**Целевая аудитория:** студенты и преподаватели российских университетов (B2B-лицензия для вуза), а также частные пользователи.

**Технологический стек:** HTML5 / CSS3 / JavaScript (фронтенд), Supabase + PostgreSQL (бэкенд и БД), Row Level Security (безопасность данных), GitHub (версионный контроль), Figma (дизайн), WEEEK (управление проектом), Qwen, Miro.

## 2. Основной функционал

Система должна обеспечивать следующие ключевые возможности:

* **Авторизация и онбординг:**
    * Регистрация и вход по email / паролю через `Supabase Auth`.
    * Трехуровневый онбординг с автоматическим разветвлением: частный пользователь / студент / преподаватель / администратор вуза.
* **Профили пользователей:**
    * Анкета с двумя списками навыков: «Чему могу учить» и «Чему хочу научиться».
    * Для университетских пользователей — привязка к вузу, институту, курсу и группе.
* **Поиск и мэтчинг:**
    * Поиск менторов по навыкам, факультету и курсу.
    * Публичный рейтинг менторов на основе оценок уроков.
* **Уроки:**
    * Запись на урок, проведение через стороннюю видео-ссылку (Zoom, Telegram).
    * Оценка урока кнопками «+» / «-» по завершении; автоматическое обновление рейтинга.
    * Автоматическое уведомление-напоминание за 30 минут до начала.
* **Экономика Skill Coins:**
    * При регистрации частный пользователь получает 3 стартовых коина.
    * За урок: ученик платит 5 Skill Coins, ментор получает 3 (2 Skill Coins — налог платформы).
    * Баланс Skill Coins отображается в шапке сайта постоянно.
* **P2P-чат:**
    * Текстовая переписка между пользователями внутри платформы.
* **Панель администратора вуза:**
    * Мониторинг активности и выгрузка отчетов.
    * Полный доступ к данным платформы в рамках своего вуза через RLS.

## 4. Приоритизация требований (MoSCoW)

### 4.1 Must Have — войдет в MVP обязательно
Без этих функций продукт не выполняет базовую ценность и не может быть принят:
* Регистрация и авторизация по email (`FR-01`, `FR-02`)
* Трехуровневый онбординг с ролями (`FR-03`, `FR-04`)
* Профиль с двумя списками навыков и привязкой к вузу (`FR-06`, `FR-07`)
* Быстрый выбор навыков кликами из каталога (`FR-13`)
* Поиск менторов по навыку и фильтр по факультету/курсу (`FR-10`, `FR-11`)
* Запись на урок и проведение через внешнюю ссылку (`FR-14`)
* Оценка урока «+» / «-» и обновление рейтинга (`FR-15`, `FR-24`)
* Стартовый баланс 3 Skill Coins для частного пользователя (`FR-18`)
* Транзакция за урок: 5 / 3 Skill Coins с налогом 2 (`FR-19`)
* Отображение баланса в шапке (`FR-20`)
* P2P-текстовый чат (`FR-27`)
* Панель администратора вуза (`FR-27`)
* RLS-изоляция данных пользователей (`FR-29`)

### 4.2 Should Have — желательно в MVP
Важные функции, улучшающие UX и монетизацию, но не блокирующие запуск:
* Загрузка аватара (`FR-08`)
* Сортировка результатов поиска по рейтингу (`FR-12`)
* Штраф за пропуск урока (`FR-25`)
* Интерфейс (заглушка) покупки пакетов Skill Coins (`FR-21`)
* Интерфейс (заглушка) Premium-подписки (`FR-23`)

### 4.3 Could Have — можно добавить после MVP
Желательные функции для повышения вовлеченности и виральности:
* Вход через Google / VK ID (`FR-05`)
* Онлайн-статус пользователя (`FR-09`)
* Уведомление о появлении нового ментора по навыку (`FR-30`)
* Бонус за развернутый текстовый отзыв (`FR-27`)
* Предпросмотр / демо-сессия (`FR-17`)

### 4.4 Won't Have — за рамками текущего релиза
Отложено до следующих версий продукта:
* Запись видео-урока с согласия сторон (`FR-18`)
* Встроенные WebRTC видео-комнаты *(запланировано на осень 2026 г.)*
* Мобильные приложения iOS / Android *(запланировано на 2027 г.)*
* ИИ-мэтчинг на основе истории обучения *(запланировано на 2027 г.)*
* Модуль комьюнити / форум *(запланировано на конец 2026 г.)*
* Интеграция с университетскими информационными системами

## 6. Критерии приемки

Функция считается принятой, если выполнены все перечисленные ниже условия для соответствующего требования:

| Требования | Функция | Критерий приемки |
| :--- | :--- | :--- |
| **FR-01/02** | Регистрация и вход | Новый пользователь успешно регистрируется, получает подтверждение и попадает в онбординг. Повторный вход по email/паролю открывает дашборд без ошибок. |
| **FR-03** | Онбординг (3 уровня) | Пользователь типа "частный" пропускает академический блок. Студент выбирает университет, институт, группу. Администратор видит только экран токена верификации. |
| **FR-06/07** | Профиль | В профиле заполнены оба списка навыков. У студента указаны университет, институт, группа. Данные сохраняются после перезагрузки страницы. |
| **FR-10/11** | Поиск | Поиск по навыку возвращает релевантных менторов. Фильтр по факультету / курсу корректно сужает выборку. |
| **FR-14** | Запись на урок | Ученик видит кнопку записи на урок, запрос отображается у ментора. Оба пользователя получают уведомление. |
| **FR-15** | Оценка урока | После завершения урока обе стороны видят кнопки "+" и "-". Выбор обновляет публичный рейтинг пользователей. |
| **FR-20/21** | Экономика коинов | При регистрации частному пользователю начисляются 3 Skill Coins. При завершении урока баланс ученика уменьшается на 5, ментора — увеличивается на 3. |
| **FR-28** | P2P-чат | Два пользователя могут обменяться сообщениями в реальном времени. История сохраняется. |
| **FR-29** | Панель администратора | Администратор входит по токену, видит статистику активности и может скачать отчет. |
| **FR-30** | Безопасность (RLS) | Попытка изменить чужой профиль возвращает ошибку доступа. Обычный пользователь не видит чужих транзакций. |

### Общие критерии для любой функции:
* Функция работает стабильно в актуальных версиях Chrome, Firefox, Safari.
* Интерфейс отображается корректно на мобильных устройствах (ширина от 320 px).
* Отсутствуют ошибки в консоли браузера, связанные с реализованной функцией.
* Данные в Supabase соответствуют ожидаемому состоянию после действия пользователя.

## 7. Definition of Done (DoD)

Задача считается полностью завершенной (Done) при выполнении всех пунктов ниже:

### 7.1 Разработка
* Функциональность реализована в соответствии с описанием требования.
* Код размещен в GitHub-репозитории проекта в релизной ветке.
* Код прошел ревью хотя бы одним членом команды.
* Отсутствуют критические ошибки в консоли браузера при использовании функции.

### 7.2 Тестирование
* Функция протестирована вручную по всем сценариям из критериев приемки.
* Позитивный сценарий проходит без ошибок.
* Негативный сценарий (некорректный ввод, отсутствие данных) обрабатывается корректно.
* RLS-политики проверены: пользователь не получает доступ к чужим данным.
* Тестирование проведено на актуальных версиях Chrome и мобильном браузере.

### 7.3 UX и дизайн
* Реализация соответствует Hi-Fi макетам в Figma.
* Интерфейс адаптирован для мобильных устройств (ширина от 320 px).
* Пользователь может выполнить целевое действие без дополнительных инструкций.

### 7.4 Данные и безопасность
* Структура таблиц в Supabase соответствуют спроектированной схеме БД.
* Триггеры и автоматизации (начисление Skill Coins, создание профиля) протестированы.
* Политики RLS настроены и проверены для всех затронутых таблиц.
* Видеопоток не сохраняется; персональные данные не передаются третьим лицам.

### 7.5 Документация
* README в репозитории обновлен (при необходимости) с учетом новой функциональности.
* Задача переведена в статус Done в таск-трекере WEEEK.
* Если функция затрагивает архитектуру — обновлена схема в Miro.

> **Важно:** Если хотя бы один пункт DoD не выполнен — задача возвращается в работу.
